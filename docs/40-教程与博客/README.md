# 教程与博客

本目录用于沉淀**里程碑级**教程、技术博客或至少先行整理的大纲。

## 使用约定

- 每个 `Issue` 结束后，优先写对应开发日志，放在 `docs/30-开发日志/`
- 每个**里程碑**结束后，结合该阶段提交、日志与关键设计决策，整理：
  - 教程正文，或
  - 技术博客正文，或
  - 至少一份可继续扩写的大纲

## 推荐命名

- `M2-鉴权与角色-教程大纲.md`
- `M3-双语内容模型-技术博客.md`
- `M4-AI-Provider-适配器-教程大纲.md`

## 最低要求

- 即使当前内容还不够完整，也要先产出大纲
- 大纲至少包含：
  - 背景与问题
  - 为什么这样设计
  - 本里程碑拆解过程
  - 关键提交与关键文件
  - 踩坑与取舍
  - 后续可延展章节
