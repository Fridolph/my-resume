## 当前系列结构

当前这组文档的作用，不是一次性写完所有长文，而是先把每个阶段收成：

- 这轮要解决什么问题
- 为什么这样拆 issue
- 当前刻意不做什么
- 后续最自然的衔接方向是什么

这样后续写正式教程或博客时，就不会只剩零散代码和提交记录。

## 与开发日志的关系

- 里程碑教程：放在当前目录
- issue 级开发日志：放在 `docs/30-开发日志/`

推荐用法：

1. 先读当前目录里的里程碑大纲
2. 再去 `docs/30-开发日志/` 找对应 issue 的实现细节
