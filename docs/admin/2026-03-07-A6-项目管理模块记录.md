# Admin A6：项目管理模块记录

## 背景

A5 完成后，后台已经开始承接简历内容，但前台项目页仍然主要依赖 SDK mock 数据。虽然 `W4` 与 `W5` 已经把前台项目展示做成了结构化内容驱动，但如果后台没有项目管理模块，前台项目页依然缺少真正的内容源头。

因此，A6 的核心任务很明确：建立后台项目管理模块，把项目列表、项目说明、标签、排序、多语言内容和发布状态都统一收口到后台，形成前台项目展示的上游管理入口。

## 本阶段目标

A6 聚焦以下问题：

- 建立项目列表页。
- 建立项目创建与编辑表单。
- 支持项目排序。
- 支持项目标签与技术栈维护。
- 支持项目多语言说明。
- 支持项目外链和封面字段。
- 支持项目发布状态管理。

这一阶段完成后，后台就不只是能管理简历和文案，而是能够继续向前台项目页提供稳定的数据结构基础。

## 为什么 A6 继续采用统一项目文档模型

A6 延续了 A5 的思路，没有把项目拆成很多散落字段或多张彼此独立的状态，而是先建立统一的项目文档模型。这么做的原因很直接。

项目内容本身天然包含多维属性：

- 列表排序
- 发布状态
- 标签与技术栈
- 封面图
- 外链
- 多语言标题、描述和摘要

如果这些能力在一开始就拆得过散，后续接数据库、接版本流和接前台读取层时，很容易出现字段边界不清晰的问题。统一模型更适合作为后台项目管理第一阶段的稳定承载面。

## 本次实现

### 1. 在共享类型中定义项目管理模型

A6 在 `packages/types` 中新增了：

- `ProjectLocaleContent`
- `ProjectRecord`

其中：

- 文档级负责 `slug`、`status`、`sortOrder`、`cover`、`externalUrl`、`tags`、`updatedAt`
- locale 级负责 `title`、`description`、`summary`

这样设计以后，后台项目管理与前台项目读取之间就具备了清晰的共享契约基础。

### 2. 建立 `useProjectManagement` composable

A6 新增了 `apps/admin/app/composables/useProjectManagement.ts`，统一管理项目数据和交互逻辑。这个 composable 负责：

- 项目列表状态
- 关键字筛选
- 当前编辑语言
- 状态筛选
- 当前选中项目
- 新建项目
- 保存项目
- 项目排序
- 标签维护
- 发布状态切换
- 项目删除
- 多语言覆盖统计

这样页面层就不需要自己维护复杂的列表操作和编辑逻辑，而是只负责展示和编排。

### 3. 建立项目管理页面

A6 新增了 `apps/admin/app/pages/projects/index.vue`，作为项目管理主页面。页面包含以下区域：

- 顶部模块说明与新建按钮
- 项目统计卡片
- 查询、语言与状态筛选区
- 项目列表区
- 多语言覆盖概览区
- 项目编辑区

这个页面继续沿用前几个后台模块的通用组织方式：

- 用 `UCard` 做功能分区
- 用筛选区统一承载查询和切换状态
- 用 toast 给出操作反馈
- 用显式按钮完成排序、编辑、删除等动作

这样可以保持后台模块之间一致的交互体验。

### 4. 支持项目排序

A6 没有只停留在“能编辑字段”，而是补上了项目排序能力。当前通过“上移 / 下移”操作维护 `sortOrder`，用于验证后台对前台项目展示顺序的控制能力。

这一步很重要，因为前台项目列表并不只是一个普通表单结果，它往往需要运营可控的展示顺序。A6 先把这个能力在本地管理状态中跑通，后续接真实存储时会更顺畅。

### 5. 支持标签、外链与封面字段

A6 把项目的扩展信息也纳入了同一个编辑区，包括：

- 标签 / 技术栈
- 外链 URL
- 封面图链接

这意味着当前项目管理模块已经具备了为前台项目卡片和详情页补充更完整结构的能力，而不是只维护一个标题和描述。

### 6. 支持多语言说明与发布状态

A6 延续当前后台的多语言组织方式：以 locale 为维度管理完整项目说明，而不是把每个字段拆成离散翻译项。当前支持：

- `zh-CN`
- `en-US`

同时，项目继续复用统一的 `PublishStatus` 模型：

- `draft`
- `reviewing`
- `published`
- `archived`

这使得项目内容不只是“可编辑”，还已经具备了进入正式内容工作流的基础条件。

### 7. 更新后台导航与首页入口

A6 完成后，后台导航已经补上“项目”入口，后台首页也增加了项目管理卡片。这样项目管理就和用户、文案、简历模块一起，正式进入后台第一阶段的业务版图中。

## 前后台对接价值

A6 的意义不仅在后台本身，还在于它正在为前台项目展示建立更真实的内容来源方向。

当前前台项目页已经具备：

- 项目列表展示
- 项目详情页展示
- 多语言内容读取入口

而 A6 刚好把后台项目结构整理为：

- 排序可控
- 状态可控
- locale 可控
- 标签可控
- 封面与外链可控

这意味着后续如果要把前台项目页从 SDK mock 切换为后台项目内容，数据边界已经比之前更接近真实接入条件。

## 本地验收结果

### 1. 类型检查

执行通过：

- `pnpm --dir apps/admin typecheck`
- `pnpm --dir apps/web typecheck`

### 2. 本地服务

执行：

- `pnpm --dir apps/admin dev --port 3001`

由于本地 `3001` 端口被占用，Nuxt 自动切换到 `3002`，最终在 `http://localhost:3002` 完成验收。

### 3. 鉴权与页面访问验证

已验证：

- 未登录访问 `/projects` 会被重定向到 `/login?redirect=/projects`
- 注入管理员 session cookie 后，首页可展示“项目管理”入口
- 注入管理员 session cookie 后，`/projects` 页面可正常访问

并确认页面已经返回以下关键内容：

- `A6`
- `项目管理`
- `查询、语言与状态`
- `多语言覆盖概览`
- `Resume Platform`
- `Design System Draft`
- `i18n Content Hub`

这说明 A6 模块已经进入后台路由、鉴权和页面编排链路，而不是停留在孤立状态。
