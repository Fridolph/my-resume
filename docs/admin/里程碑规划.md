# Admin 重构里程碑规划

## 文档说明

这份文档用于跟踪管理后台的重构过程。它的目标是让后台从一开始就有明确的边界、稳定的实施节奏和持续更新的过程记录。

使用方式如下：

- 每个里程碑开始前，确认本阶段范围。
- 每个子任务完成后，填写“完成总结”。
- 方案调整时，在“变更记录”中补充说明。
- 每完成一个里程碑，再进入下一个里程碑。

---

## Admin 重构目标

新的 `admin` 应用承担以下职责：

- 作为内部管理后台。
- 管理用户、角色与权限。
- 管理国际化文案。
- 管理简历内容、项目内容与站点配置。
- 为前端展示站提供稳定的内容管理入口。
