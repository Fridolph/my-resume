# Admin A7：站点配置与收尾记录

## 背景

A6 完成后，后台已经具备了用户、文案、简历和项目这几个核心业务模块，但仍然缺少一个站点级收口层。换句话说，后台虽然可以维护具体内容，却还没有统一管理默认语言、社交链接、下载资源和 SEO 默认配置的能力。

如果这一层缺席，后台第一阶段就仍然是不完整的。因为对于一个公开内容平台来说，站点级配置和业务内容一样重要，它们决定了整个站点的默认行为和对外呈现方式。

因此，A7 的任务不是再扩展新的业务模块，而是把后台第一阶段真正收尾：补齐站点设置，并把前面几个模块的边界整理成一个闭环。

## 本阶段目标

A7 聚焦以下问题：

- 建立站点配置页面。
- 管理默认语言。
- 管理社交链接配置。
- 管理下载资源配置。
- 管理 SEO 默认配置。
- 完成后台第一阶段收尾。

这一阶段完成后，后台第一阶段就不再只是分散的管理页面集合，而是开始具备完整的“业务模块 + 站点配置”结构。

## 为什么 A7 先采用统一站点配置模型

A7 没有把站点设置拆成多个分散页面，而是先建立一个统一的站点配置模型。这么做有两个直接好处。

第一，默认语言、社交链接、下载资源和 SEO 默认配置本来就都属于站点级能力，它们应该放在同一层管理。第二，后续如果要接数据库或接配置中心，也更适合从一个统一的配置对象出发，而不是把配置散落在不同页面和不同状态里。

因此，A7 采用了统一配置记录的方式，把站点级内容集中管理，并通过单一 composable 暴露给设置页使用。

## 本次实现

### 1. 在共享类型中定义站点配置模型

A7 在 `packages/types` 中新增了：

- `SiteSocialLink`
- `SiteDownloadLink`
- `SiteSeoDefaults`
- `SiteSettingsRecord`

这些类型把站点级配置正式纳入共享层，也为后续前后台对接、配置持久化和 Web 端消费站点设置打下了结构基础。

### 2. 建立 `useSiteSettingsManagement` composable

A7 新增了 `apps/admin/app/composables/useSiteSettingsManagement.ts`，用于统一管理站点设置相关的状态和交互逻辑，包括：

- 默认语言切换
- 社交链接增删
- 下载资源增删
- SEO 默认配置维护
- 更新时间维护
- 保存校验

这样设置页本身就不需要承担太多状态处理逻辑，而是继续保持“页面编排层”的职责。

### 3. 重建站点设置页面

A7 将原本只是权限占位的 `apps/admin/app/pages/settings/index.vue` 重构为完整的设置页。当前页面包含以下几个功能区：

- 默认语言与阶段说明
- 社交链接配置
- 下载资源配置
- SEO 默认配置

这意味着 `settings` 页面已经从“权限验证页”升级为真正可承载后台第一阶段站点级管理能力的页面。

### 4. 管理默认语言

A7 在设置页中增加了默认语言切换能力，当前支持：

- `zh-CN`
- `en-US`

这个能力虽然看起来简单，但它是前后台内容系统联动的关键配置之一。后续无论前台默认语言如何决定，还是后台内容录入以哪个语言为主，这个设置都会成为一个稳定的站点级基础项。

### 5. 管理社交链接与下载资源

A7 补上了两个非常实用的站点级内容区块：

- 社交链接配置
- 下载资源配置

这样后台不仅能维护业务内容，还能维护公开站点中“联系我”“资源下载”这一类高频站点配置。对于个人内容平台来说，这两个区块是非常有现实价值的。

### 6. 管理 SEO 默认配置

A7 在设置页中增加了默认 SEO 配置管理，包括：

- 站点标题
- 站点域名
- 默认描述
- 默认 OG 图地址

这也让 A7 和前面刚完成的 `W6` 形成了自然衔接：Web 端已经有了 SEO 能力，而后台开始具备了管理这些默认信息的结构能力。

### 7. 完成后台第一阶段收尾

A7 完成后，还顺手更新了：

- 后台导航阶段标识
- 后台首页阶段说明
- 设置页的阶段说明文案

这意味着后台第一阶段已经从功能建设进入阶段收尾状态，整个后台应用的模块边界也已经比之前更清晰。

## 后台第一阶段结果

A7 完成后，后台第一阶段已经具备以下模块：

- 登录与权限框架
- 用户管理
- 文案管理
- 简历管理
- 项目管理
- 站点配置

从架构上看，后台第一阶段已经形成了一个比较完整的内容管理后台雏形：既能管理具体业务内容，也能管理站点级配置。

## 本地验收结果

### 1. 类型检查

执行通过：

- `pnpm --dir apps/admin typecheck`

### 2. 本地服务

执行：

- `pnpm --dir apps/admin dev --port 3001`

由于本地 `3001` 端口被占用，Nuxt 自动切换到 `3002`，最终在 `http://localhost:3002` 完成验收。

已验证：

- 带管理员 session cookie 的首页可以返回 `A7` 阶段内容
- 未登录访问 `/settings` 会被重定向到 `/login?redirect=/settings`
- 带管理员 session cookie 后，`/settings` 页面可以正常访问

并确认页面命中以下关键内容：

- `A7`
- `站点设置`
- `默认语言与阶段说明`
- `社交链接配置`
- `下载资源配置`
- `SEO 默认配置`

这说明 A7 模块已经进入后台权限链路和页面编排链路，而不是停留在占位阶段。
