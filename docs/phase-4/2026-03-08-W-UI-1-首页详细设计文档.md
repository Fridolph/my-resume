# W-UI-1：首页详细设计文档（规划稿）

## 一、任务定位

`W-UI-1` 是第四阶段的起点，它不是单纯做一个首页改版，而是要先回答一个更重要的问题：

> 当前这个首页，究竟是一个“简历入口页”，还是一个“个人 IP 与产品化能力的总入口页”？

如果只是做简历入口页，那么首页只需要：

- 写清楚你是谁
- 放一些技能标签
- 链接到简历和项目

但如果首页要承担更长期的价值，它就不能停留在“个人信息陈列页”，而应该逐步成为：

- 个人 IP 的第一触点
- 能力与作品的总入口
- 未来内容扩展、服务表达、产品承接的统一门面

所以 `W-UI-1` 的首页设计，不应该只围绕“好不好看”，而要围绕：

- 首页到底要让用户记住什么
- 首页如何在第一屏建立信任与辨识度
- 首页如何把“你这个人”与“你正在做的东西”连接起来

## 二、首页在整个站点中的角色

下一阶段的首页，建议承担四个角色。

### 1. 个人身份入口

首页首先要清楚地告诉用户：

- 你是谁
- 你主要做什么
- 你现在处于什么方向上

这里不是传统简历式“姓名 + 岗位”那么简单，而是要形成更清晰的个人定位表达。

### 2. 能力证明入口

首页不应该只说“我会什么”，而应该快速给出：

- 你做过什么类型的事情
- 你擅长解决什么问题
- 你的工作方法和系统能力是什么

这意味着首页需要把“技能”升级为“能力证明”。

### 3. 内容与作品入口

首页要把站内最重要的内容入口清楚地组织出来，让用户自然进入：

- 简历页
- 项目页
- 后续可能扩展的内容页 / 专栏 / 产品页

这会让首页更像一个“信息中枢”，而不是孤立页面。

### 4. 未来产品化入口

虽然当前阶段不在文档中展开商业细节，但从公开站点设计角度，首页应该预留未来继续演进的空间，例如：

- 内容体系扩展
- 服务能力展示
- 产品能力展示
- 联系与合作入口强化

所以首页应该是一个可以持续演进的产品首页，而不只是当前版本的简历封面。

## 三、首页设计的核心问题

在具体设计前，建议先明确首页要解决的三个核心问题。

### 问题 1：用户进入首页后，3 秒内应该知道什么？

建议 3 秒内至少知道：

- 这是 Fridolph 的个人站点 / 个人内容入口
- 方向是前端、全栈、内容系统、产品化建设相关
- 这里不只是简历，还有项目与系统化成果

### 问题 2：用户继续下滑时，应该逐步建立什么认知？

建议逐步建立的认知是：

- 你不仅会写页面
- 你有系统搭建能力
- 你有内容与产品思维
- 你做的事情是有方法、有结构、可持续扩展的

### 问题 3：用户离开首页前，应该被引导做什么？

建议首页至少要成功引导用户进入以下之一：

- 查看简历
- 查看项目
- 建立“这个人值得继续了解”的印象

如果后续继续扩展，也可以加入：

- 联系合作
- 查看方法论内容
- 查看产品化方向内容

## 四、首页主叙事建议

首页不建议继续使用传统“自我介绍式简历首页”的叙事方式，而建议使用：

- 个人身份
- 当前方向
- 系统能力
- 代表性成果
- 明确入口

这样的层层递进叙事。

### 建议叙事框架

可以把首页理解为五个连续问题。

#### 第一层：你是谁？

这是身份建立层。

需要解决：

- 用户第一次进入页面时的陌生感
- 个人品牌的基础识别

这里建议表达：

- 姓名 / 英文名 / 个人品牌名
- 当前角色定位
- 一句足够明确的主张

#### 第二层：你正在做什么？

这是方向建立层。

需要解决：

- 用户知道你不是只会写代码
- 用户知道你关注的是更完整的系统和产品表达

这里建议表达：

- 当前聚焦方向
- 你搭建的内容平台 / 系统化工程方向
- 你如何把前端、后台、内容、产品思考串起来

#### 第三层：你的能力如何被证明？

这是信任建立层。

这里不建议只列技术栈，而建议给出更“结果导向”的能力标签，比如：

- 架构与工程化
- 内容系统与后台建设
- 前台展示与产品表达
- 多语言与可持续迭代能力

### 第四层：我可以看什么？

这是内容入口层。

首页要明确给出：

- 简历入口
- 项目入口
- 未来可扩展入口

入口不只是按钮，而应该带有解释，让用户知道进去后能看到什么。

### 第五层：为什么值得继续关注？

这是 IP 感建立层。

首页不一定要直接营销自己，但应该让用户感受到：

- 你不是把站点当作一次性作品
- 你正在长期建设一套东西
- 这里未来还会继续成长

这个感觉，会让首页从“个人页面”更接近“个人品牌产品入口”。

## 五、首页信息架构建议

建议首页采用以下结构。

## 1. 顶部导航区

### 目标

让站点入口足够简洁清楚，同时保留现在已有的：

- 首页
- 简历
- 项目
- 语言切换
- 主题切换

### 设计建议

- 导航保持简洁，不宜过重
- 左侧强化站点名 / 个人标识
- 右侧保留关键导航与切换入口
- 可以考虑增加一个更清晰的 CTA，如“查看项目”或“了解我”

### 视觉方向

- 轻量
- 克制
- 固定但不压迫内容

## 2. Hero 首屏区

这是首页最关键的一屏，建议投入最多心力。

### 目标

让用户在第一屏就建立：

- 身份认知
- 专业认知
- 持续浏览意愿

### 建议内容结构

#### A. 品牌徽标 / Badge

用于表达当前站点的身份属性，例如：

- Personal Operating System
- Builder / Designer / Engineer
- Personal Content Platform
- 个人技术与内容入口

这个 badge 的作用不是装饰，而是提前给用户一个理解框架。

#### B. 主标题

主标题不建议只是“Fridolph Web”，而建议更清楚地表达：

- 你的身份
- 你的方向
- 你的价值主张

例如方向上应该接近：

- 我在构建一个结合前端、内容系统与产品表达的个人平台
- 我把工程、内容和展示统一成一个长期演进的个人入口

不一定按这个句子写，但要有这种层次。

#### C. 副标题 / Supporting Copy

副标题负责补全主标题没有说完的部分，例如：

- 当前关注的技术与产品方向
- 为什么这个站点存在
- 它不只是一个简历页，而是一个正在生长的系统入口

#### D. 主要 CTA + 次级 CTA

建议首页首屏保留两个 CTA：

- 主 CTA：查看项目
- 次 CTA：查看简历

原因是：

- 项目更容易体现“做成了什么”
- 简历更适合作为结构化补充

如果把“简历”放成第一 CTA，会更像传统求职站；把“项目”放在前面，会更像产品化个人站。

#### E. 侧边辅助信息 / 证明块

Hero 区可以加入一个辅助信息块，用于快速证明：

- 已有项目数量
- 内容系统状态
- 多语言支持
- 当前建设方向

但这里不建议只用冷冰冰的统计数字，而应该尽量让它们带有“含义”。

例如不是：

- 2 个项目
- 2 种语言

而是更像：

- 已有公开案例
- 多语言内容体系已运行
- 后台与公开内容已打通

这会让首页更有“产品建设进度感”。

## 3. 能力证明区

### 目标

让用户看到：

- 你做的不只是某一个页面
- 你具备系统化能力

### 建议结构

建议不是单纯列技能栈，而是做成 3~4 个能力卡片，每个卡片表达一个能力方向，例如：

- 工程化与架构
- 后台与内容系统
- 前台展示与体验设计
- 多语言与持续迭代

### 每张卡片建议内容

- 一个清晰标题
- 一段解释这项能力“解决什么问题”的短描述
- 1~2 个辅助证明点

## 4. 核心入口区

### 目标

把首页真正变成站点入口，而不是停留页。

### 建议入口

- 简历：看系统化履历与能力结构
- 项目：看代表性成果与案例
- 后续预留：文章 / 方法论 / 产品 / 实验项目

### 设计建议

入口卡片应该更像“可继续探索的内容模块”，而不是简单按钮。

每个入口建议包含：

- 标题
- 一句话解释
- 明确行动词

## 5. 当前建设状态 / 长期感表达区

### 目标

这是“个人 IP 化”很重要的一层。

首页除了展示“我现在有什么”，还建议表达：

- 我正在持续建设什么
- 这不是一次性的个人页面
- 这是一个长期演进中的个人系统

### 呈现方式建议

可以是一个轻量 section，例如：

- 当前站点建设阶段
- 当前内容系统已打通哪些部分
- 下一阶段会继续优化什么

这一块会让用户感受到：

- 你在持续做事
- 这个站点是 alive 的
- 它有成长中的产品气质

## 六、视觉表现建议

## 1. 总体气质

首页建议采用：

- 高留白
- 强层级
- 轻装饰
- 内容驱动
- 局部强调

而不是：

- 密集堆信息
- 过强拟物
- 复杂营销组件感

## 2. 页面节奏建议

建议首页有明显的节奏变化：

- 首屏强表达
- 中段能力证明
- 再往下内容入口
- 最后长期感与收口

不要让所有 section 视觉权重完全平均，否则首页会显得没有主次。

## 3. 卡片策略建议

卡片不应该只是统一复制，而应该按用途分层：

- Hero 证明卡片
- 能力卡片
- 内容入口卡片
- 状态说明卡片

它们应该属于同一体系，但不应该完全同一个样子。

## 4. 配色建议

建议保持当前整体调性，但可以加强：

- 主强调色
- 高亮文字色
- 卡片边界层级
- 背景区分层次

让页面整体看起来更“有设计”但不浮夸。

## 七、内容文案方向建议

如果首页要更有个人 IP 感，文案就不能只是传统简历式写法。

### 不建议的方向

- 纯岗位描述式标题
- 纯技术关键词堆叠
- 只有“我会什么”没有“我做成了什么”

### 建议的方向

- 更像一个长期建设者的表达
- 更强调系统能力与连接能力
- 更突出你正在做的事情，而不是只列你会什么

换句话说，首页文案应该从：

- 求职式表达

逐步转向：

- 个人品牌式表达
- 建设者式表达
- 产品入口式表达

## 八、首页成功标准建议

当 `W-UI-1` 完成后，首页应至少达到以下状态：

### 1. 第一眼更强

用户打开首页后，会更容易留下印象，而不是只看到“一个技术人站点”。

### 2. 主叙事更清楚

用户能更快理解你是谁、在做什么、为什么值得继续浏览。

### 3. 入口更自然
n用户从首页进入简历页或项目页的动机会更强，不需要靠简单导航硬跳。

### 4. 个人 IP 感更明确

首页会更像一个“个人品牌产品入口”，而不只是“个人简历首页”。

## 九、本任务建议拆分（供后续编码前继续确认）

如果你确认进入 `W-UI-1` 的实现阶段，建议继续拆成以下子任务：

- `W-UI-1-1`：首页信息架构与主叙事确认
- `W-UI-1-2`：首页视觉基线与 section 体系设计
- `W-UI-1-3`：Hero 区详细设计
- `W-UI-1-4`：能力证明区与入口区详细设计
- `W-UI-1-5`：首页细节收口与响应式设计
- `W-UI-1-6`：首页实现记录与总结文档

## 十、结论

`W-UI-1` 的首页，不应只是一个更漂亮的首页，而应是：

- 个人身份表达页
- 能力证明页
- 内容导航页
- 未来产品化入口页

如果这一层设计成立，后面的简历页、项目页和详情页就不再是零散页面，而是会被一个更明确的个人 IP 叙事串起来。

下一步如果你认可这份详细设计方向，我建议继续只在本地推进：

- 首页分区草图级方案
- 首页文案策略稿
- `W-UI-1-1 / W-UI-1-2` 的进一步细化文档
