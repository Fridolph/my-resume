# 第四阶段执行里程碑（规划稿）

## 说明

第四阶段建立在第三阶段已经完成的三端联通、后台协作增强与测试基线之上，当前阶段默认继续沿用以下正式技术主线：

- `apps/web`：Nuxt 4 展示端
- `apps/admin`：Nuxt 4 管理后台前端
- `apps/api-server`：NestJS API Server
- 数据层：`SQLite + Drizzle ORM`
- 组件与前端基础：`Nuxt UI + TypeScript`

第四阶段不再继续扩张底层架构，也不优先推进部署与运维，而是聚焦：

- Web 展示层视觉优化
- 页面内容表现升级
- 前台统一展示规范沉淀
- 响应式与主题一致性提升
- 阶段总结与下一阶段入口准备

## W-UI-1：前台视觉基线与首页改版

### 目标

先建立前台统一视觉基线，并优先改造首页，让公开站点的第一印象、主叙事和浏览引导明显提升。

### 子任务

- [ ] W-UI-1-1：梳理首页信息架构与主叙事层级
- [ ] W-UI-1-2：定义前台视觉基线与核心样式方向
- [ ] W-UI-1-3：重构首页 Hero 区与首屏层次
- [ ] W-UI-1-4：优化首页统计卡片与模块入口表现
- [ ] W-UI-1-5：收口首页响应式与主题表现
- [ ] W-UI-1-6：整理首页优化记录与阶段总结

### 完成标准

- 首页第一屏更有主次和识别度
- 用户能更快理解站点定位与核心内容
- 首页卡片、区块与 CTA 表现更统一
- 首页在明暗主题与常见屏宽下表现稳定

## W-UI-2：简历页视觉升级

### 目标

提升简历页的阅读节奏、结构层级和专业展示感，让简历内容从“结构可读”进一步走向“表达更强”。

### 子任务

- [ ] W-UI-2-1：梳理简历页信息层级与阅读路径
- [ ] W-UI-2-2：重构顶部个人信息与关键信息区
- [ ] W-UI-2-3：优化教育 / 经历 / 技能 / 联系方式模块节奏
- [ ] W-UI-2-4：强化时序信息、重点信息与标签表现
- [ ] W-UI-2-5：优化多语言排版与移动端适配
- [ ] W-UI-2-6：整理简历页优化记录与阶段总结

### 完成标准

- 简历页重点更突出、阅读更顺畅
- 多模块内容之间层级更清楚
- 多语言切换后排版稳定、信息不混乱
- 简历页更像展示页而不是简单信息列表

## W-UI-3：项目列表页视觉升级

### 目标

把项目列表页从“项目数据展示页”提升为更有作品感和浏览吸引力的公开作品列表页。

### 子任务

- [ ] W-UI-3-1：梳理项目列表页卡片结构与展示重点
- [ ] W-UI-3-2：重构项目卡片的标题、摘要、标签与操作入口
- [ ] W-UI-3-3：优化列表页节奏、分区与留白关系
- [ ] W-UI-3-4：补强 hover / focus / active 等交互状态
- [ ] W-UI-3-5：优化项目列表在主题与响应式下的一致性
- [ ] W-UI-3-6：整理项目列表页优化记录与阶段总结

### 完成标准

- 项目卡片更具辨识度和层次感
- 项目列表更像作品集而不是后台输出结果
- 标签、摘要和入口信息更易读
- 列表页在不同设备上的浏览体验更自然

## W-UI-4：项目详情页结构升级

### 目标

把当前偏过渡态的项目详情页升级为更具案例表达能力的项目讲述页，为后续更完整项目内容扩展打基础。

### 子任务

- [ ] W-UI-4-1：梳理项目详情页目标结构与叙事顺序
- [ ] W-UI-4-2：重构详情页顶部概览区与关键信息区
- [ ] W-UI-4-3：设计背景 / 目标 / 实现 / 结果等内容容器
- [ ] W-UI-4-4：优化返回入口、上下文导航与辅助信息表现
- [ ] W-UI-4-5：为后续后台项目详情扩展预留展示位
- [ ] W-UI-4-6：整理项目详情页优化记录与阶段总结

### 完成标准

- 项目详情页不再只是结构占位页
- 页面能够承载更完整的案例表达
- 当前详情页结构具备继续扩展的空间
- 项目详情页与项目列表页的视觉语言基本统一

## W-UI-5：视觉统一收口与多端适配

### 目标

统一收口第四阶段前面页面改造的视觉语言，并补齐响应式、主题和细节表现上的一致性。

### 子任务

- [ ] W-UI-5-1：统一前台通用 section / card / heading 表现
- [ ] W-UI-5-2：统一标签、按钮、辅助文案与状态表现
- [ ] W-UI-5-3：收口暗黑模式与明亮模式的一致性
- [ ] W-UI-5-4：收口移动端与中大屏适配细节
- [ ] W-UI-5-5：整理前台视觉规范与实现约束文档
- [ ] W-UI-5-6：完成第四阶段视觉收口记录

### 完成标准

- 首页、简历页、项目页、详情页视觉语言统一
- 明暗主题之间切换自然，不出现明显风格断裂
- 多端适配表现稳定
- 前台视觉规范有文档可沉淀、可复用

## W-UI-6：第四阶段总结与下一阶段准备

### 目标

完成第四阶段沉淀，总结视觉优化成果、边界、风险和后续演进方向。

### 子任务

- [ ] W-UI-6-1：整理第四阶段总结文档
- [ ] W-UI-6-2：整理第四阶段交付清单
- [ ] W-UI-6-3：整理当前视觉规范、风险与待办
- [ ] W-UI-6-4：更新下一阶段规划入口

### 完成标准

- 第四阶段有正式总结与交付清单
- 当前前台视觉能力边界清晰
- 后续迭代方向有明确入口

## 当前建议执行顺序

结合当前阶段目标，建议按以下顺序推进：

1. 先完成 `W-UI-1`：前台视觉基线与首页改版
2. 再推进 `W-UI-2`：简历页视觉升级
3. 然后完成 `W-UI-3`：项目列表页视觉升级
4. 再推进 `W-UI-4`：项目详情页结构升级
5. 最后完成 `W-UI-5` 与 `W-UI-6`：统一收口与阶段总结
