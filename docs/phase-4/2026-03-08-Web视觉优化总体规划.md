# Web 视觉优化总体规划（规划稿）

## 说明

这份文档用于承接第三阶段完成后的下一阶段方向：

- Web 展示层界面与内容呈现优化

当前文档只做：

- 目标梳理
- 设计方向规划
- 页面级优化建议
- 阶段拆分建议

当前文档暂时不做：

- 具体代码实现
- 组件直接改造
- 视觉稿 1:1 落地
- 动效与图片资源正式接入

也就是说，这一版是“下一阶段实施前的规划稿”，等确认后再进入详细设计和编码阶段。

## 一、为什么现在进入 Web 视觉优化阶段

前面几个阶段，项目已经完成了更偏底层和中台能力的建设：

- Monorepo 架构已稳定
- `apps/web` / `apps/admin` / `apps/api-server` 三端已打通
- `SQLite + Drizzle ORM + NestJS` 的内容链路已跑通
- Admin 后台已经具备真实登录、权限、内容管理、版本、发布能力
- Web / Admin / API 已有基础自动化验证入口

这意味着当前项目已经不再缺“骨架”，而是开始进入更需要提升“表现层”的阶段。

换句话说，现在最值得继续投入的，不再是：

- 再补一个底层模块
- 再打通一条数据链路

而是：

- 让公开站点看起来更像一个完整产品
- 让内容表达更有层次、更有辨识度
- 让当前已经具备的数据能力，在视觉上真正被体现出来

所以，下一阶段进入 Web 视觉优化，是当前最自然也最有价值的方向。

## 二、当前 Web 的主要问题

从当前公开站点的状态看，Web 已经具备：

- 首页
- 简历页
- 项目列表页
- 项目详情页
- 基础 SEO
- 多语言与主题切换入口

但整体仍然偏“结构可用”，距离“成熟展示站”还有明显差距。

### 1. 页面有结构，但品牌感偏弱

当前页面虽然已经能展示内容，但整体视觉风格还不够鲜明，主要问题包括：

- 首页的第一印象不够强
- 视觉层级偏平
- 缺少明确的个人品牌感
- 页面之间的情绪调性还没有真正建立起来

### 2. 内容能看，但缺少重点引导

当前页面里的很多信息已经具备了，但用户在浏览时，未必能快速抓到重点。例如：

- 首页 Hero 区的主叙事不够集中
- 简历页模块虽然完整，但重点不够聚焦
- 项目列表与详情页的“作品感”还不够强

也就是说，内容已经存在，但视觉表达还没有把内容价值放大。

### 3. 页面之间还缺少统一的展示语言

当前 Web 虽然使用了统一技术栈和组件体系，但从用户视角看：

- 页面节奏不够统一
- 卡片层次不够稳定
- 标题、描述、辅助信息的视觉权重还不够明确
- 各页面的内容展示逻辑还没有完全沉淀成统一规则

## 三、下一阶段的总体目标

下一阶段不追求“花哨改版”，而是希望把当前 Web 提升为一个：

- 更有产品感
- 更有品牌感
- 更适合长期扩展
- 更适合公开展示与商业演进

的展示端。

更具体地说，下一阶段希望实现四个目标。

### 1. 建立更明确的视觉主叙事

让用户打开首页时，能更快理解：

- 这是什么项目
- 这个人/这个站点的核心定位是什么
- 当前站点最值得继续浏览的内容是什么

### 2. 提升内容表达的层次感

让现在已经存在的数据，不只是“被渲染出来”，而是：

- 被更有节奏地组织
- 被更明确地强调重点
- 被更自然地引导阅读

### 3. 沉淀前台统一展示规范

让首页、简历页、项目页、项目详情页不再是“分别成立的页面”，而是逐步形成统一的展示系统，包括：

- 页面节奏
- 区块组织方式
- 标题体系
- 卡片体系
- 标签与状态表现方式

### 4. 为后续继续商业化和产品化留出视觉扩展空间

当前阶段不会把公开文档写成商业规划，但从工程和设计角度来看，下一阶段的视觉优化应具备：

- 可持续扩展
- 可继续品牌化
- 可继续演进为更完整展示产品

的基础。

## 四、视觉方向建议

这一阶段建议不是做“完全重做”，而是在保留当前 Nuxt + Nuxt UI 结构优势的前提下，逐步建立更统一的视觉语言。

### 1. 总体风格建议：克制、现代、内容优先

建议的总风格方向：

- 克制
- 现代
- 偏内容型
- 偏个人品牌展示
- 兼顾产品感与专业感

不建议的方向：

- 过度装饰
- 过度依赖炫技动效
- 过度营销落地页化
- 与当前技术内容调性不匹配的视觉堆砌

### 2. 视觉关键词建议

下一阶段视觉关键词可以围绕：

- 清晰
- 节奏
- 层次
- 专业
- 可信
- 轻产品化

### 3. 页面气质建议

页面气质上更适合接近：

- 个人品牌站
- 技术型产品展示站
- 作品与履历结合的内容站

而不是单纯的：

- 简单简历页
- 传统博客首页
- 纯营销页

## 五、页面级优化建议

## 1. 首页优化方向

首页应该承担三个任务：

- 建立第一印象
- 快速说明核心身份与能力范围
- 引导进入简历和项目两个核心模块

### 当前问题

- Hero 区力度不够
- 统计卡片较平
- 模块入口表达偏“功能性”
- 页面节奏略平均，缺少主次

### 优化建议

- 强化 Hero 区主标题、副标题、主 CTA
- 调整统计卡片为更强的“证明型信息”
- 优化模块入口卡片，让简历 / 项目 / 多语言能力更具可读性
- 增加更明显的内容分区节奏
- 适当加入头像、签名、品牌标识或个人主视觉元素（如后续需要）

### 预期结果

首页在优化后应当让用户更快感知：

- 你是谁
- 你做什么
- 为什么值得继续看

## 2. 简历页优化方向

简历页的目标不是只把结构化数据列出来，而是让履历内容更有“叙事性”和“专业展示感”。

### 当前问题

- 模块之间视觉差异还不够大
- 阅读节奏略平均
- 重点内容没有充分突出
- 页面更像结构化信息列表，而不是经过设计的履历展示页

### 优化建议

- 强化顶部个人信息区的视觉权重
- 优化教育 / 经历 / 技能 / 联系方式四类模块的层级关系
- 对核心标题、关键标签、时序信息做更明显的视觉强化
- 适当引入时间线、分组标题、强调色点缀等方式增强履历感
- 优化中英文切换后的排版稳定性

### 预期结果

简历页优化后应达到：

- 信息多但不乱
- 重点清楚
- 更适合招聘、合作、介绍场景下的快速浏览

## 3. 项目列表页优化方向

项目列表页是当前 Web 最适合体现“作品感”的页面之一。

### 当前问题

- 卡片表达偏朴素
- 项目之间差异感不够强
- 标签和摘要虽然有，但视觉辨识度不够
- 页面仍然更偏后台内容输出视角，而不是对外展示视角

### 优化建议

- 强化项目卡片层次感
- 更好地组织标题、描述、标签、摘要、操作入口
- 如果后续有封面资源，可逐步把项目封面纳入主视觉
- 提升卡片 hover / focus 态一致性
- 让项目列表更接近作品陈列而不是简单表单化内容块

### 预期结果

项目页应成为一个：

- 可快速浏览
- 有辨识度
- 更像“作品集”而不是“项目表”

的公开展示页。

## 4. 项目详情页优化方向

项目详情页目前还是过渡态，下一阶段非常值得重点优化。

### 当前问题

- 当前详情页仍偏 DTO / 结构展示感
- 对项目的价值表达不足
- 缺少项目背景、问题、方案、结果等更完整的展示节奏

### 优化建议

- 将详情页从“过渡信息页”升级为“项目讲述页”
- 建立更明确的详情页区块，如：
  - 项目概览
  - 技术栈
  - 背景与目标
  - 实现思路
  - 亮点与结果
- 优化返回入口与上下文导航
- 为后续继续接入后台项目详情结构预留展示容器

### 预期结果

项目详情页应逐步具备真正的案例页潜力，而不只是简单的详情占位页。

## 六、设计系统与前台规范建议

这一阶段除了页面级优化，还建议逐步补前台的视觉规范沉淀。

### 1. 建议补一套前台主题规范

建议至少明确：

- 主色 / 强调色
- 文本层级颜色
- 边框与分割线层级
- 页面背景层次
- 卡片阴影与圆角策略

### 2. 建议收口通用前台组件样式

可以逐步统一：

- 页面 Intro 头部区
- 数据卡片
- 内容区块卡片
- 标签系统
- CTA 按钮样式
- 页面容器与 section 间距

### 3. 建议建立页面节奏规范

例如可以逐步定义：

- 顶部首屏信息密度
- section 间距标准
- 标题与描述之间的节奏
- 大屏 / 中屏 / 移动端的适配优先级

## 七、建议的实施顺序

如果你确认进入下一阶段，建议按下面顺序推进。

### 第一组：视觉基线与首页优先

先做：

- 前台视觉基线整理
- 首页 Hero 与首页整体节奏优化
- 通用 section / card / heading 表现统一

原因是：首页会最先决定整个站点的视觉基调。

### 第二组：简历页与项目列表页

接着做：

- 简历页重构
- 项目列表页重构

因为这两个页面是公开站点最核心的内容承载页。

### 第三组：项目详情页深化

然后做：

- 项目详情页结构升级
- 项目案例叙事能力增强
- 为后续内容扩展预留模板能力

### 第四组：统一细节收口

最后做：

- 响应式细节
- 动效细节
- 暗黑模式一致性
- 多语言排版稳定性
- 页面间视觉统一检查

## 八、阶段拆分建议（供后续确认）

下一阶段可以考虑拆成如下里程碑。

### W-UI-1：前台视觉基线与首页改版

建议范围：

- 定义视觉基线
- 首页首屏与首页模块节奏优化
- 通用卡片与 section 表现统一

### W-UI-2：简历页视觉升级

建议范围：

- 简历信息层次优化
- 履历模块节奏优化
- 多语言排版优化

### W-UI-3：项目列表页视觉升级

建议范围：

- 项目卡片重构
- 标签与摘要表现优化
- 列表页节奏和交互状态优化

### W-UI-4：项目详情页结构升级

建议范围：

- 项目详情页内容结构升级
- 从 DTO 页过渡到案例页表达
- 为后续更完整项目内容预留扩展位

### W-UI-5：视觉统一收口与多端适配

建议范围：

- 响应式细节
- 暗黑模式统一
- 动效与交互细节
- 页面间视觉一致性检查

## 九、确认前建议

在正式开始下一阶段编码前，建议你先确认三件事：

### 1. 是否保留当前“克制、现代、内容优先”的视觉方向

如果需要更强品牌感或更强商业感，可以在下一版规划里进一步收紧风格关键词。

### 2. 是否优先首页，再推进内页

当前我建议：

- 首页优先
- 再简历 / 项目列表
- 最后项目详情

### 3. 是否需要在进入编码前先补一版更细的页面级详细设计

如果你希望后面执行更稳，下一步可以先不编码，而是继续出：

- 首页详细设计文档
- 简历页详细设计文档
- 项目列表页详细设计文档
- 项目详情页详细设计文档

## 十、结论

当前阶段非常适合开始 Web 视觉优化，但这一步不应该理解为“简单美化”，而应该理解为：

- 让当前已经打通的内容系统真正被更好地展示
- 让公开站点开始具备更成熟的产品感与品牌感
- 为后续长期迭代留下更稳定的前台视觉规范

如果你认可这份总体规划，下一步我建议先继续输出：

- 下一阶段里程碑规划
- 首页详细设计文档

然后再正式进入编码阶段。
