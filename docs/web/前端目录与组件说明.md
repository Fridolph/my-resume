# Web 前端目录与组件说明

## 目标

这份文档用于说明 `apps/web` 当前第一阶段的目录边界、核心 composable 和展示组件职责，方便后续继续重构时快速判断“应该把代码放在哪里”。

## 目录边界

当前 `apps/web/app` 采用以下分层：

- `pages/`：页面编排层，只负责路由页面、页面级数据绑定和 SEO 调用。
- `components/web/`：展示组件层，负责复用 UI 结构，不承载页面级请求逻辑。
- `composables/`：站点级状态、内容读取和 SEO 能力封装。

这种分层的目的很简单：

- 页面不直接堆叠大段展示细节。
- 展示组件不直接耦合请求和状态源。
- 可复用逻辑尽量收口到 composable。

## 页面说明

当前页面包括：

- `pages/index.vue`：首页
- `pages/resume.vue`：简历页
- `pages/projects/index.vue`：项目列表页
- `pages/projects/[slug].vue`：项目详情页

它们都遵循同一模式：

- 通过 `useWebContent` 读取数据
- 通过 `usePageSeo` 设置页面 SEO
- 通过 `Web*` 组件完成展示拼装

## 展示组件说明

当前主要展示组件包括：

- `WebPageContainer`：统一页面宽度与留白
- `WebSectionIntro`：统一页面介绍区
- `WebStatCard`：统计信息卡片
- `WebFeatureCard`：首页功能入口卡片
- `WebTagList`：标签与技术栈展示
- `WebResumeBlockCard`：简历章节卡片
- `WebProjectPreviewCard`：项目预览卡片
- `WebNavigation`：头部导航
- `WebLocaleSwitch`：语言切换入口
- `WebThemeSwitch`：主题切换入口
- `WebFooter`：站点页脚与阶段说明

这些组件共同承担“稳定输出展示结构”的职责，不直接处理内容源和业务状态。

## Composable 说明

当前主要 composable 包括：

- `useWebContent`：统一前台内容读取入口
- `useWebLocale`：语言状态与 UI 文案管理
- `useWebTheme`：主题状态与持久化管理
- `usePageSeo`：页面级 SEO 能力封装
- `useSiteSeoConfig`：站点级 SEO 配置读取

后续如果继续扩展 Web 端，应优先判断新增逻辑属于哪一层，再决定是放进页面、组件还是 composable。

## 后续约束建议

后续继续开发时，建议保持以下规则：

- 页面层不直接写大量卡片结构，优先复用 `components/web/`
- 页面层不直接拼装复杂站点状态，优先收口到 composable
- 展示组件不直接请求数据
- 站点级配置优先从统一 composable 或共享配置读取

这样可以避免第一阶段已经整理好的边界被后续迭代重新打乱。
