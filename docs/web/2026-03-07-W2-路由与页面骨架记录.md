# Web W2：路由与页面骨架建立记录

## 背景

W1 结束后，`web` 已经具备 Nuxt 4 工程骨架，但它仍然只是一个“能打开的展示站壳”。如果不尽快把页面结构和路由骨架建立起来，后续无论是接入简历内容、项目数据还是 SEO 能力，都会缺少稳定的落点。

因此 W2 的任务是把 `web` 从单页骨架升级为真正的站点骨架。

## 本阶段目标

W2 主要解决以下问题：

- 建立公开站点的页面层级。
- 建立统一导航和页脚。
- 建立首页、简历页、项目列表页和项目详情页的骨架。
- 建立页面级 SEO 封装入口。
- 为后续内容接入阶段保留稳定的页面边界。

## 为什么 W2 先做页面骨架

如果在页面结构还不稳定时就直接接内容数据，会带来两个问题：

- 页面边界很容易随着数据接入过程反复变化。
- 后续导航、SEO 和布局组件会被业务逻辑裹挟，难以抽离。

因此 W2 先把“页面长什么样、路由怎么组织、公共壳怎么复用”固定下来，再把数据接进来。这是更稳的顺序。

## 本次实现

### 1. 建立统一导航与页脚

W2 新增了两个站点级组件：

- `WebNavigation`
- `WebFooter`

它们统一承载站点导航与页脚说明，使 `app.vue` 从简单的页面壳升级为真正的站点壳。

这样处理后，后续页面无论怎样扩展，都不需要重复维护站点公共结构。

### 2. 建立页面级 SEO composable

W2 新增了 `usePageSeo`，用于统一设置页面的：

- `title`
- `description`
- `ogTitle`
- `ogDescription`
- `ogUrl`

这一步虽然看起来简单，但非常重要。它让页面 SEO 从一开始就走统一入口，而不是分散在每个页面里各写一套逻辑。

### 3. 建立首页正式结构

W2 重写了首页，使首页不再只是 W1 的工程说明页，而是升级为真正的站点入口页。

当前首页已经具备：

- 页面级介绍区
- 面向简历、项目和平台方向的入口卡片
- 统一的 SEO 信息

### 4. 建立简历页骨架

W2 新增了 `/resume` 页面。当前它先以章节占位方式验证页面结构，后续会接入：

- 基础信息
- 教育经历
- 工作经历
- 技能亮点
- 联系方式

这样做的好处是，后面在接内容时不会再去反复改动页面层级。

### 5. 建立项目列表页骨架

W2 新增了 `/projects` 页面，用于承载项目卡片列表。当前它先使用静态占位数据验证：

- 项目卡片结构
- 列表到详情页的跳转关系
- 页面 SEO 封装

### 6. 建立项目详情动态路由

W2 新增了 `/projects/[slug]` 动态路由页面。这一步很关键，因为它标志着 `web` 已经不再是单页展示站，而是具备了内容站最基础的详情页承载能力。

当前详情页先不接真实数据，只验证以下边界：

- 动态参数读取
- 页面标题与描述拼装
- 返回列表页的导航路径

## 本地验收结果

W2 不仅通过了类型检查，也实际跑通了本地页面访问。

### 1. 类型检查

执行通过：

- `pnpm --dir apps/web typecheck`

### 2. 本地服务

执行：

- `pnpm --dir apps/web dev --port 3000`

本地展示站服务已成功启动。

### 3. HTTP 验证

已验证以下页面：

- `/` 返回 `200 OK`
- `/resume` 返回 `200 OK`
- `/projects` 返回 `200 OK`
- `/projects/resume-platform` 返回 `200 OK`

并验证了页面内关键文案已经正确返回，说明页面骨架已真实生效，而不是仅仅创建了文件。

## 当前结果

W2 完成后，`web` 已具备以下能力：

- 统一站点布局
- 统一导航与页脚
- 首页骨架
- 简历页骨架
- 项目列表页骨架
- 项目详情动态路由骨架
- 页面级 SEO 统一封装
