# Web W3：展示组件体系重建记录

## 背景

W2 解决了站点路由和页面骨架的问题，但页面内部仍然存在大量“页面自己拼卡片”的写法。如果继续在这种基础上接入真实内容，后续很容易出现两个问题：

- 同类 UI 在不同页面里重复实现。
- 页面组件逐渐承担太多展示细节，难以维护。

因此 W3 的目标很明确：把 Web 端从“页面骨架”推进到“展示组件体系”。

## 本阶段目标

W3 主要解决以下问题：

- 建立可复用的页面容器。
- 建立可复用的统计卡片和功能卡片。
- 建立可复用的标签组组件。
- 建立简历章节卡片和项目预览卡片。
- 让首页、简历页、项目页改为由组件拼装，而不是直接写页面细节。

## 为什么 W3 要先抽展示组件

如果页面骨架刚稳定就直接去接内容，很容易把“页面结构”和“展示细节”一起锁死。后面一旦想统一视觉或复用区块，就会反复回头改页面。

因此 W3 先把展示层抽出来，有两个直接收益：

- 后续内容接入时，页面只负责组合组件和绑定数据。
- 后续文案管理接入时，组件边界更清楚，不会把文案散落在页面各处。

## 本次实现

### 1. 建立页面容器组件

W3 新增了 `WebPageContainer`，统一页面的宽度、留白和内容区块节奏。这样首页、简历页、项目页和详情页不再各自维护布局间距。

### 2. 建立统计卡片与功能卡片

W3 新增了：

- `WebStatCard`
- `WebFeatureCard`

它们分别用于承载：

- 页面阶段信息
- 入口卡片信息

这样首页和详情页中的信息展示不再依赖临时 `UCard` 结构，而是有了统一表达方式。

### 3. 建立标签组组件

W3 新增了 `WebTagList`，统一展示技术栈、关键词和高亮标签。这个组件后续不仅能用于项目页，也能用于简历页的技能展示。

### 4. 建立简历章节卡片

W3 新增了 `WebResumeBlockCard`，用来承载简历章节信息。这样简历页不再直接在页面里写一堆重复卡片，而是由结构化数组驱动组件渲染。

### 5. 建立项目预览卡片

W3 新增了 `WebProjectPreviewCard`，用来统一项目列表页中的项目卡片。当前它已经支持：

- 标题
- 描述
- 标签组
- 跳转详情页

这为后续接入后台项目数据打下了基础。

### 6. 页面重构为组件拼装

W3 完成后，以下页面都已经切换到组件拼装模式：

- 首页
- 简历页
- 项目列表页
- 项目详情页

这意味着页面层现在更像编排层，而不是细节层，这正是后续持续重构需要的状态。
