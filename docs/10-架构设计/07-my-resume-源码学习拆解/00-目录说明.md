# my-resume：源码学习拆解目录说明

## 一、为什么增加这一组文档

这一组文档不是面向“功能完成记录”，而是面向“源码理解与接棒”。

它的目标是把 `my-resume` 当前已经跑起来的三条主线继续拆清楚：

- `server`：唯一业务后端、鉴权、简历持久化、AI / RAG
- `admin`：后台前端壳、会话、资源缓存、简历编辑、AI 工作台
- `web`：公开站展示、已发布简历读取、展示组件渲染链路

这样后续再继续源码讲解时，就不会散落在聊天记录里，而是统一沉淀到这一组目录中。

## 二、当前目录约定

### `01-server-后端与-AI-RAG-链路索引.md`

- 作为 `server` 学习线的索引入口
- 当前先指向已经整理完成的后端 / AI / RAG 文档

### `02-admin-后台前端与简历编辑链路.md`

- 记录 `admin` 端前端壳、登录态、资源缓存、简历编辑器的源码理解
- 后续继续补：
  - `ResumeDraftEditorPanel`
  - `AI 工作台`
  - `resource store / session store`
  - 动态加载与前端架构拆分

### `03-web-公开站渲染链路.md`

- 记录 `web` 端公开页、已发布简历读取、组件渲染与展示结构
- 后续继续补：
  - 首页数据读取
  - summary / full snapshot 区分
  - 展示组件分层
  - header / locale / section 渲染

## 三、后续使用原则

- 后续源码梳理优先继续写到本目录
- `server / admin / web` 三条线尽量分开，不再混写
- 若某次讲解跨越多端，也优先在各自文档中分别补充，再在需要时增加总览索引

## 四、建议阅读顺序

1. `01-server-后端与-AI-RAG-链路索引.md`
2. `02-admin-后台前端与简历编辑链路.md`
3. `03-web-公开站渲染链路.md`

如果你当前主要想补前端理解，建议先看：

1. `02-admin-后台前端与简历编辑链路.md`
2. `03-web-公开站渲染链路.md`
