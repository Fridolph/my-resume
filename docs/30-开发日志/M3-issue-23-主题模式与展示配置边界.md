# M3-issue-23-主题模式与展示配置边界

## 标题

- Issue：`#37`
- 里程碑：`M3 简历内容与发布流`
- 分支：`docs/m3-issue-23-theme-boundaries`
- 日期：`2026-03-25`

## 背景

简历站天然是内容展示型产品，哪怕 `v1` 先不做多模板，`light / dark` 也属于基础体验能力。如果不在当前阶段先约束清楚，后续很容易把颜色、布局、主题判断直接散落在页面和组件里，导致后面难以扩展。

## 本次目标

- 明确 `light / dark` 属于 `v1` 基础能力。
- 明确多模板主题当前不实现，但要预留扩展钩子。
- 明确内容模型与展示配置分离。

## 非目标

- 不实现真实主题切换。
- 不实现多模板系统。
- 不修改现有 `StandardResume` 内容模型。

## 实际改动

- 更新架构总方案，补充主题策略与展示扩展钩子约束。
- 更新研发流程，补充展示层任务在 Review / 自测时的主题检查项。
- 更新里程碑与 Issue 拆解建议，新增 `issue-23` 规划项。

## Review 记录

- 当前改动只落在文档与流程层，没有提前实现 UI 能力，符合教程节奏。
- 主题扩展点放在展示层，而不是内容层，避免后续结构污染。

## 自测结果

- 人工检查通过：术语与现有 `M3 / M5` 规划一致。
- 人工检查通过：没有引入与当前里程碑无关的实现性改动。

## 可沉淀为教程/博客的点

- 为什么展示型项目要尽早把主题能力定为架构约束。
- 为什么内容模型和展示配置应当分离。

## 后续待办

- 后续为多模板简历展示补充注册式扩展设计。
