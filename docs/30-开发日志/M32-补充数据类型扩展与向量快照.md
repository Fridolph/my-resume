# M32 补充数据类型扩展 + 向量快照流程 — 开发日志

## 背景

my-resume 的 RAG 补充知识库（user_docs）仅支持 hobby/tech_blog/knowledge_column/general 四种类型。实际使用中发现工作经历的补充说明（如行业背景、项目细节、岗位职责）也有强烈的 RAG 检索需求，需要新增类型支持。

## 本次目标

1. 新增 `work_detail` contentType，映射到 experience 知识域
2. Admin UI 同步适配
3. 完善向量快照（snapshot）导出/加载流程的日志与文档

## 设计决策

**为什么不在 admin 单独建一个"工作经历"分类？**

现有 user_docs 体系已有完整的 contentType → knowledgeDomain → renderHint 映射链。work_detail 天然映射到 experience 域，与简历核心经历共享同一检索域。用户问工作相关问题 → 路由自动命中 experience → resume_core + work_detail 联合召回。如果单独建分类，需要独立建表、独立检索逻辑，增加维护成本。
