# M22 里程碑收束：AI 简历导入识别与 API 契约

## 一、里程碑目标回顾

M22 的核心目标：
1. **AI 简历导入识别**：上传中文简历 → AI 识别为 StandardResume 候选草稿 → diff 看台 → 按模块回填
2. **API 响应契约标准化**：统一全局响应信封 + Swagger 注解
3. **构建与部署增强**：版本追踪、启动日志、数据库路径规范化

## 三、关键架构

### 简历导入识别（12 轮演进）

```
上传简历 → 异步 Job 状态机（10阶段）→ AI 生成候选草稿
→ repair → normalize → validate 防线
→ 模块级 diff 看台 → 用户确认 → 按模块回填 draft
```

完整时序与源码演进见：
- `docs/60-源码拆解/06-M22-AI-简历导入识别-时序图与优化路线.md`
- `docs/60-源码拆解/07-M22-简历导入识别-12轮源码演进.md`

### API 契约标准化

统一 `{ code, message, data }` 响应信封，Controller 通过 `@ApiEnvelopeResponse` 装饰器声明，`SkipResponseEnvelope` 装饰器用于 SSE/文件下载等特殊场景。
