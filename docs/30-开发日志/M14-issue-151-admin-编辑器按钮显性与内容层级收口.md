## 背景

后台简历编辑器已经具备完整编辑能力，但删除按钮在密集布局里辨识度偏弱，从教育经历开始的模块也存在明显的卡片套卡片感，影响长表单的浏览效率。

## 本次目标

- 提高删除按钮的显性、点击感和危险操作反馈。
- 从教育经历开始，对内容模块做轻量去层级化收口。
- 保持现有双语编辑、折叠逻辑和保存链路不变。

## 非目标

- 不重写 `ResumeDraftEditorPanel` 的整体布局。
- 不调整基础信息模块结构。
- 不新增服务端接口或数据字段。

## TDD / 测试设计

- 保持 `resume-draft-editor-panel` 既有保存测试继续通过。
- 为新增的教育/技能/亮点删除按钮补显性断言，确认仍可通过 `aria-label` 查询。

## 实际改动

- 放大删除图标尺寸与线条粗细，并将删除按钮提升为更明显的 danger icon button。
- 为 `EditorEntry` 增加轻量 `embedded` 视觉变体，用于教育、工作、项目、技能、亮点模块。
- 将项目链接子卡片收口为更轻的 `editor-subcard`，降低内容嵌套感。
- 在 `globals.css` 中补充 entry shell、body 和 subcard 的轻量样式。

## Review 记录

- 改动只触达 admin 展示与交互层，没有改变字段、状态管理和保存契约。
- 删除按钮的热区和折叠按钮保持同级尺寸，避免长表单里操作目标不一致。

## 自测结果

- `pnpm --filter @my-resume/admin test` ✅
- `pnpm --filter @my-resume/admin typecheck` ✅

## 遇到的问题

- 当前删除按钮其实热区不小，真正问题主要是图标太细、颜色过轻、和折叠按钮主次接近。
- 最终通过图标升级、按钮尺寸提升和 danger hover 态一起收口，而不是单独放大图标。

## 可沉淀为教程/博客的点

- 组件变体如何帮助我们在不改业务逻辑的前提下压低视觉层级。
