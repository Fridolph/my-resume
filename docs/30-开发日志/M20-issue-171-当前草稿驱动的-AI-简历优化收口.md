# M20 issue-171：当前草稿驱动的 AI 简历优化收口

## 背景

- 现有 AI 工作台已经具备“读取当前草稿 → 生成结构化建议 → 选择模块 → 应用回草稿”的基础链路。
- 但前端主路径仍偏“粘贴文本 / 文件提取 → 分析”，不够像一个真正围绕当前简历工作的优化工作台。

## 本次目标

- 将 AI 工作台的简历优化主路径收口为“当前草稿优化”。
- 用户只需要输入目标岗位、JD 或优化要求，即可基于 DB 当前草稿生成结构化 diff。
- 保留辅助分析报告与文件提取能力，但降级为辅助入口。

## 实际改动

- 重构 AI 工作台表单结构：
  - 增加“当前草稿基线”提示区。
  - 将文本输入语义改为“目标岗位 / JD / 优化要求”。
  - 将“分析当前草稿并生成建议”改成主 CTA。
  - 将“生成辅助分析报告”收口为次级路径。
- 调整 diff 面板文案：
  - 使用“当前草稿 vs AI 建议稿”表达。
  - 强化“影响模块”“写回当前草稿”等措辞。
- 同步更新 AI 工作台页面描述、模块 README 和相关测试。

## Review 记录

- 复查现有服务端能力后确认，本轮不需要改 `resume-optimize` 与 `resume-optimize/apply` 的服务端核心逻辑。
- M20 只做产品路径和工作台交互语义收口，不进入编辑器内联建议层。

## 遇到的问题

- 现有工作台测试较多围绕旧按钮文案和“分析输入”措辞，需要同步更新断言。
- AI 工作台既要保留缓存报告与真实分析，又要突出“当前草稿优化”为主路径，需通过表单分区降低混淆。

## 测试与验证

- 计划执行 admin AI 工作台相关单测与构建验证：
  - `analysis-panel.spec.tsx`
  - `ai-workbench-shell.spec.tsx`
  - `pnpm --filter @my-resume/admin build`
