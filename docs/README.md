## 当前文档分层

### `10-架构设计`

- 记录项目目标、技术选型、架构原则
- 适合先看“为什么这样设计”

### `20-研发流程`

- 记录 GitHub 开发流程、里程碑拆解、测试策略与质量门禁
- 适合先看“我们如何一步步做”

### `30-开发日志`

- 每个 issue 一篇开发日志
- 适合回看具体实现过程、review 记录和踩坑

### `40-教程与博客`

- 每个里程碑至少一份教程大纲
- 适合从阶段角度回顾项目是怎样逐步长出来的

### `50-教程文章`

- 面向外部读者的正文版本
- 适合按里程碑连续阅读“为什么这样做、如何一步步复现”

### `60-源码拆解`

- 面向仓库内部学习与讲解
- 适合按“全景图 + 真实调用链 + 分层原因”来理解源码

## 当前推荐阅读主线

如果你的目标是理解这个项目如何从静态简历站重构到教程型 monorepo，可以按这个节奏读：

1. 总方案与边界
2. GitHub 标准开发流程
3. 开源版发布前检查清单
4. M2 ~ M10 的教程大纲
5. `docs/50-教程文章/` 里的正文教程
6. `docs/60-源码拆解/` 里的源码讲解
7. 你感兴趣阶段的开发日志
