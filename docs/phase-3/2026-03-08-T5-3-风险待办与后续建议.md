# T5-3：风险待办与后续建议

## 当前已知风险

### 1. 部署与可观测性仍未进入正式实施

当前第三阶段虽然已经完成 `T3` / `T4` / `T5`，但：

- 生产部署流程
- 数据持久化上线方案
- 日志与运行巡检
- 异常追踪与线上问题排查

仍然属于后续需要回补的部分。

这并不是遗漏，而是当前阶段的主动取舍。

### 2. 当前 E2E 仍然是 smoke 级覆盖

虽然现在已经拥有 Web / Admin 的 Playwright 测试，但它们当前仍然主要覆盖：

- 关键路径是否可达
- 关键模块是否渲染
- 少量关键交互是否存在

后续如果要继续提升质量，还可以逐步补充：

- 多语言切换验证
- 更多权限角色矩阵
- 关键表单提交流程
- 更细的发布前后内容校验

### 3. lint 基线仍存在历史遗留问题

当前项目已经具备类型检查、E2E 和 API 测试，但全量 lint 仍然有历史遗留项尚未系统收口。

这意味着后续如果要继续提升工程一致性，可以单独开一个阶段，对 lint 和格式基线做专项治理。

## 当前待办建议

结合当前阶段状态，后续建议优先关注：

- 回补 `T1` / `T2` 的部署与可观测性
- 在下一阶段继续增强 Web 展示层视觉与内容呈现
- 逐步扩展 E2E 覆盖深度
- 逐步提升 lint / 规范统一程度

## 建议的后续优先级

### 高优先级

- 正式部署与环境收口
- 运行健康与故障排查入口
- Web 展示层视觉和内容呈现升级

### 中优先级

- 更细粒度的角色矩阵测试
- 更丰富的公开站点内容交互测试
- 发布流程与回滚流程进一步文档化

### 低优先级

- 更复杂的 diff 视图
- 更重型的协作审批机制
- 更完整的内容操作审计视图
