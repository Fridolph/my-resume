# T1：正式部署落地任务拆分

## 目标

把当前三端项目从“具备部署蓝图”推进到“具备真实部署落地能力”，并形成可复用的部署与回滚手册。

## T1-1：确定部署目标与运行拓扑

### 目标

明确第三阶段采用的目标部署平台、服务拓扑、域名关系与持久化方案。

### 关注点

- 选择哪一种部署平台作为当前阶段落地目标
- Web / Admin / API 是三服务分离还是部分合并
- SQLite 数据文件如何持久化
- 域名与反向代理如何组织

### 产出

- 一份部署拓扑设计文档
- 一份目标平台选型说明

## T1-2：补齐部署环境变量与生产配置

### 目标

让三端在生产环境所需的配置项清晰可控。

### 关注点

- Web 公开域名
- Admin API 地址
- API CORS 白名单
- 数据库路径
- 生产模式下的默认值与约束

### 产出

- 环境变量文档更新
- 部署配置文档更新

## T1-3：完成 API Server 部署与数据库持久化

### 目标

先把 API Server 与数据库链路放到目标运行环境中稳定运行。

### 关注点

- API 服务启动
- 数据库路径挂载
- migration 执行
- 健康检查可访问
- 备份策略可执行

### 产出

- API Server 部署记录
- 数据库持久化与备份说明

## T1-4：完成 Web / Admin 部署与域名接入

### 目标

让 Web 与 Admin 都能在目标环境通过正式地址访问。

### 关注点

- Web 域名访问
- Admin 域名访问
- API 请求地址正确
- 站点 SEO 地址正确
- 反向代理关系正确

### 产出

- Web / Admin 部署记录
- 域名接入记录

## T1-5：整理部署手册与回滚手册

### 目标

把本轮部署过程整理为后续可重复执行的标准手册。

### 关注点

- 首次部署步骤
- 更新部署步骤
- 回滚步骤
- 常见故障与排查顺序

### 产出

- 部署操作手册
- 回滚操作手册

## T1-6：完成部署后联调与验收记录

### 目标

部署完成后，对三端主链路做一次正式验收。

### 关注点

- Web 首页、简历、项目页可访问
- Admin 登录与关键模块可访问
- API 健康检查与公开接口可访问
- 发布链路与公开内容读取正常

### 产出

- 部署后联调记录
- T1 完成总结

## 建议执行顺序

建议按以下顺序推进：

1. `T1-1`：确定部署目标与运行拓扑
2. `T1-2`：补齐部署环境变量与生产配置
3. `T1-3`：完成 API Server 部署与数据库持久化
4. `T1-4`：完成 Web / Admin 部署与域名接入
5. `T1-5`：整理部署手册与回滚手册
6. `T1-6`：完成部署后联调与验收记录
