# T4-2：Admin 关键页面 E2E 冒烟记录

## 背景

`T4-1` 已经先为 Web 展示端建立了第一批 E2E 冒烟能力。接下来把同样的模式接到 Admin，是第三阶段质量体系扩展里最自然的一步。

因为对于当前项目来说，Admin 的价值不只是“页面能打开”，而是：

- 能不能完成真实登录
- 登录后能不能进入后台首页
- 内容模块能不能展示真实数据
- 第三阶段新增的版本理解和操作边界能不能在浏览器层被看到

所以 `T4-2` 的目标，是先把后台最关键的几条链路，用正式的 Playwright 冒烟测试固定下来。

## 本次实现

### 1. 为 `apps/admin` 增加 Playwright 测试基础设施

这一轮在 `apps/admin` 中新增了：

- `@playwright/test`
- `apps/admin/playwright.config.ts`
- `apps/admin/tests/smoke.spec.ts`

同时补充了脚本：

- `pnpm --dir apps/admin test:e2e`
- `pnpm --dir apps/admin test:e2e:headed`

这意味着后台页面也已经拥有和 Web 一致的正式 E2E 入口。

### 2. 使用独立 preview 端口运行 Admin 冒烟

这次为 Admin 冒烟单独使用了：

- Admin 预览端口：`3200`
- API 服务：`3011`

这样做有两个好处：

- 不会和本地正在开发的 `3002` 端口互相干扰
- 冒烟测试运行在更稳定的 preview 模式下，而不是直接依赖开发态 HMR

同时配置中继续复用了已存在的 `api-server`，让测试链路保持和真实三端架构一致。

### 3. 首批覆盖四条后台关键链路

这次首批用例覆盖了：

- 登录后可以进入后台首页
- 文案管理页可以展示版本限制提示
- 简历管理页可以展示版本历史区块
- 项目管理页可以展示排序边界提示

这些用例并不是随便挑的，而是尽量覆盖了当前后台最能代表第三阶段成果的链路：

- 真实登录态
- 首页可达
- 版本理解增强
- 操作边界增强

也就是说，这一轮测试不仅在测“页面能开”，还在测 `T3` 的核心交互成果是否真的出现在页面里。

## 本次自测

### 冒烟测试命令

以下命令已通过：

- `pnpm --dir apps/admin test:e2e`

### 通过结果

当前共通过 4 条用例：

- 登录后可以进入后台首页
- 文案管理页可以展示版本限制提示
- 简历管理页可以展示版本历史区块
- 项目管理页可以展示排序边界提示

其中浏览器级断言已经覆盖到：

- `root@fridolph.local / root123` 登录成功
- 首页欢迎信息
- 文案版本“无需重复恢复”提示
- 简历版本历史区域渲染
- 项目排序第一项禁用“上移”及其原因说明

## 本次收获

`T4-2` 的意义在于：后台的关键内容路径也终于从“人工点点看”升级成了“可以用命令重复验证”。

而且它和 `T3` 形成了非常明确的闭环：

- `T3` 负责增强页面体验
- `T4-2` 负责把这些体验增强真正纳入回归验证

这意味着后续如果有人改动后台版本区块、登录页或项目排序逻辑，已经有第一批浏览器级测试能够及时发现明显回归。
