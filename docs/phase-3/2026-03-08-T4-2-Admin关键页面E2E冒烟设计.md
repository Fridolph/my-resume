# T4-2：Admin 关键页面 E2E 冒烟设计

## 背景

`T4-1` 已经先为 Web 展示端建立了第一批可重复执行的浏览器级 smoke 测试。下一步最自然的延伸，就是把同样的思路接到 Admin 管理后台。

对于当前项目来说，Admin 的关键路径不仅涉及页面访问，还涉及：

- 登录态建立
- 权限控制后的页面访问
- 真实 API 驱动的数据展示
- 第三阶段新增的版本理解与操作边界体验

所以 `T4-2` 的目标，不是把后台所有页面和所有角色一次测完，而是先覆盖当前最关键、最能代表现阶段后台成熟度的路径。

## 本轮目标

本轮优先覆盖：

- 登录页
- 后台首页
- 文案管理页
- 简历管理页
- 项目管理页

其中重点验证：

- 真实登录链路可用
- 登录后首页可达
- 文案页版本区块可渲染
- 简历页版本区块可渲染
- 项目页排序边界与编辑入口可渲染

## 设计原则

### 1. 先测最高价值路径

Admin 当前最关键的路径，是：

- 登录
- 进入后台首页
- 访问内容模块
- 看见真实数据和关键交互区块

所以 `T4-2` 先覆盖这条链路，不在当前一轮引入更重的角色矩阵测试。

### 2. 断言优先选稳定页面元素

优先断言：

- 标题
- 页面主 heading
- 稳定按钮
- 版本区块标题
- 已经在 `T3` 中明确引入的操作提示文案

### 3. 继续使用独立 preview 端口

为了避免和本地开发端口冲突，Admin 冒烟同样使用独立 preview 端口运行，并复用 `api-server`。

## 建议实现

## 1. 为 `apps/admin` 增加 Playwright 依赖与脚本

建议增加：

- `@playwright/test`
- `test:e2e`

## 2. 新增 Admin Playwright 配置

建议配置：

- Admin 预览端口：`3200`
- API 地址：`http://127.0.0.1:3011/api/admin`
- 自动复用已运行的 `api-server`

## 3. 新增 Admin smoke 用例

建议首批覆盖：

- 登录成功后进入首页
- 文案管理页显示版本理解区块
- 简历管理页显示版本历史区块
- 项目管理页显示排序边界提示

## 完成标准

`T4-2` 完成后，应满足：

- `apps/admin` 拥有基础 Playwright 冒烟入口
- 登录与关键内容页面可浏览器级验证
- 后台关键路径具备与 Web 一致的测试收口方式
