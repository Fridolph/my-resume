# T5-4：下一阶段规划入口

## 下一阶段建议主线

基于第三阶段当前成果，下一阶段最适合继续推进的方向是：

- Web 展示层界面与内容呈现优化

原因很明确：

- 当前三端架构已经基本稳定
- 后台内容链路已经打通
- 质量体系已经拥有基础自动化入口
- Web 页面目前仍然偏“结构可用”，视觉与表现层还有明显可提升空间

## 下一阶段可拆分方向

### 1. 首页视觉层升级

可继续优化：

- Hero 区内容表现
- 统计卡片层次
- 模块区块节奏
- CTA 与信息层级

### 2. 简历页内容表现升级

可继续优化：

- 履历模块层级
- 信息密度与呼吸感
- 多语言切换体验
- 简历内容视觉强化

### 3. 项目页展示升级

可继续优化：

- 项目列表卡片表现
- 项目详情页结构
- 标签与技术栈视觉系统
- 项目封面与品牌化表达

### 4. 设计系统与视觉规范沉淀

可继续补充：

- 前台主题变量
- 组件层视觉统一
- 页面间节奏一致性
- 更适合长期扩展的展示端规范

## 下一阶段前置条件

进入下一阶段前，当前已经具备的基础包括：

- 可用的内容后台
- 可用的 API Server
- 可用的 SQLite + Drizzle 数据层
- Web / Admin / API 自动化检查入口
- 第三阶段阶段性总结文档

这意味着下一阶段可以更专注在：

- 展示效果
- 视觉表达
- 品牌感与可读性

而不需要再重复搭底层基础设施。
