# 文档导航

本文档用于规划本仓库后续的中文文档体系，便于未来接入 `VitePress` 后直接上线阅读。

## 目录规划

### `docs/00-文档导航.md`

- 文档首页
- 仓库文档阅读顺序
- 面向读者的快速导航

### `docs/10-架构设计/`

- 记录项目整体设计、技术选型、架构原则
- 适合写“为什么这样设计”

当前计划：

- `01-个人简历-monorepo-重构总方案-v1学习版.md`
- `02-为什么避免双后端.md`
- `03-文档目录规划与-VitePress-计划.md`

### `docs/20-研发流程/`

- 记录 GitHub 协作流程、里程碑管理、Issue 规范、开发日志模板
- 适合写“我们如何一步步做”

当前计划：

- `01-GitHub-标准开发流程.md`
- `02-里程碑与-Issue-拆解建议.md`
- `03-开发日志模板.md`

### `docs/30-开发日志/`

- 每个 Issue 完成后沉淀一篇开发日志
- 后续教程、博客、分享优先从这里提炼

命名建议：

- `M1-issue-01-workspace-初始化.md`
- `M1-issue-02-仓库规范与-AGENTS.md`

### `docs/40-部署上线/`

- 记录 CI/CD、Vercel、云服务器、Nginx、HTTPS 等

### `docs/50-教程文章/`

- 面向外部读者的成体系教程稿
- 可由开发日志整理后沉淀而来

## 命名规范

- 文档文件名统一使用**中文为主**
- 使用编号保证排序稳定
- 文件名建议格式：
  - `01-主题名.md`
  - `02-主题名.md`
- 若主题包含英文技术名，保留必要英文，例如：
  - `01-GitHub-标准开发流程.md`
  - `03-文档目录规划与-VitePress-计划.md`

## 写作顺序建议

- 先写架构与流程文档
- 再按 Issue 产生开发日志
- 最后从开发日志提炼教程文章

## VitePress 接入建议

- 未来增加 `docs/.vitepress/`
- 首页直接引用本目录结构
- 侧边栏按 `10/20/30/40/50` 分组即可
