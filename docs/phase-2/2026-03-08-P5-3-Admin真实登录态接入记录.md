# P5-3：Admin 真实登录态接入记录

## 背景

`P5-2` 已经把基础鉴权能力放进了 `api-server`，但如果 `admin` 还继续使用本地 mock 登录，那么这套后端鉴权能力就仍然没有进入正式主链路。

所以这一轮 `P5-3` 的核心目标非常直接：

- 管理后台登录改走真实 API
- 管理后台登出改走真实 API
- 页面守卫改为根据真实会话判断
- 页面刷新后能够恢复登录态

## 本次实现

### 1. `usePlatformApiClient` 支持 SSR 转发 cookie

本次首先改造了 `apps/admin/app/composables/usePlatformApiClient.ts`。

现在在 SSR 阶段，`admin` 会把当前请求的 `cookie` 头继续转发给 `api-server`；在浏览器端，SDK 请求则默认带上 `credentials: 'include'`。

这一步是 `P5-3` 能成立的关键，因为没有它的话，服务端渲染时就无法通过 `/api/auth/me` 恢复真实会话。

### 2. `useAuth` 改为真实 auth API 驱动

本次 `apps/admin/app/composables/useAuth.ts` 已经不再使用本地 mock 账号做登录判断。

当前链路改为：

- `login()` -> `POST /api/auth/login`
- `refreshSession()` -> `GET /api/auth/me`
- `logout()` -> `POST /api/auth/logout`

同时，前端仍然保留一份 `session state` 和 `ready state`，用于页面组件和中间件直接消费。

### 3. `auth` / `guest` 中间件改为真实会话判断

本次两个中间件都已经改为：

- 先调用 `refreshSession()`
- 再根据真实登录态决定跳转行为

这意味着：

- 未登录访问受保护页面，会被正确导向登录页
- 已登录再访问登录页，会被正确导回后台首页

### 4. 登录页切换到真实接口

本次 `apps/admin/app/pages/login.vue` 已经切换为真实登录：

- 输入邮箱密码
- 调用 `api-server` 登录接口
- 建立真实服务端 session
- 跳转回后台首页

同时，演示账号区域仍然保留，方便本地测试。

### 5. 后台外壳文案同步到新里程碑

`app.vue` 中的页头与 Footer 也同步更新到了 `P5-3` 当前阶段，便于和里程碑节奏保持一致。

## 本次实现的价值

这一轮真正改变的是：`admin` 的登录态终于不再是前端演示态，而是正式切到了后端鉴权链路。

这意味着后续继续推进：

- 接口级权限守卫
- 页面级权限控制
- 后台接口与公开接口边界

时，已经有了真实的“当前用户”基础。
